## A. 重建 Knowledge（4 条，必须手动重建）

源账号共 4 条 Knowledge Note，全部是卡卡手写规则，**新账号必须重建**。完整正文见 `knowledge/` 目录，逐条对应：

| 文件 | 名称 | 触发场景（trigger / scope） |
| --- | --- | --- |
| `knowledge/01_卡卡全局协作规则.md` | 卡卡全局协作规则 | when working on any task or project for 卡卡, including coding, planning, or communication |
| `knowledge/02_阿奇与本叔默认双角色.md` | 阿奇与本叔默认双角色 | when responding to tasks…technical execution…or strategic/content/business judgment… |

| `knowledge/04_central-memory-safe使用规则.md` | central-memory-safe 使用规则 | when using the central-memory-safe MCP tool… |

**重建方式（任选其一）：**

- **方式 1（推荐，新账号 Devin 自动做）**：在新账号开一个 Devin 会话，说一句
  > “读取本仓 `.devin/migration/knowledge/` 下每个 .md，把每条按 frontmatter 里的 name / trigger / 正文，用 `devin_knowledge_manage`（action=create）重建为 Knowledge Note。`_autogen/` 目录跳过不建。”

  Devin 会逐条调用 `devin_knowledge_manage create`（参数：`name`=frontmatter 的 name，`trigger`=frontmatter 的 scope，`content`=正文）。

- **方式 2（卡卡手动）**：在新账号后台 Knowledge 页，按每个文件的 name / trigger / 正文，逐条粘贴新建。

**注意：** `knowledge/_autogen/05_kaka1209仓库索引_autogen.md` 是**系统自动生成**的仓库索引，**不要手动重建**——新账号连上 `kaka1209` 仓后系统会自动再生成，手动建只会重复/冲突。该文件仅作存档。
