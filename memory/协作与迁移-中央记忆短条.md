# 中央记忆短条（用 remember 写入）

> 用途：在**已连 `central-memory-safe` 的账号**里，把下面这条写进中央记忆。
> 规则：`remember` 只新增、不覆盖；metadata 必须含 `type` 和 `project`。

## remember 调用

```
remember(
  content="""
【两账号 GitHub 协作 + 换账号迁移定式（kaka 项目）】
1) 容器：GitHub 组织 kaka-media。mykaloooo=owner，carolfrederic4-arch=成员。
2) 授权：组织内建团队 kaka-team，把 2026IP、devin-control 批量授权为 Write；协作者经团队拿权限，新仓加进 kaka-team 即自动继承 Write，不用逐仓加人。
3) 架构：单仓——kaka-media/2026IP 为唯一主仓；kaka-media/devin-control 存接力包（接力清单+knowledge导出+mcp/集成说明）；已废弃镜像与 MYKALOOOO_MIRROR_TOKEN。
4) 权限边界：读写代码/开PR/用Devin=成员即可；装Devin App/连组织/转仓/改组织设置/建删团队=仅owner。
5) 换账号：一个 GitHub 组织同时只能挂一个 Devin 账号，故旧账号先在 Settings→Integrations→GitHub→Disconnect，再用 owner(mykaloooo) 登录 GitHub 在新账号 Connect→选 kaka-media→All repositories→Install。
6) 设置同步：Skills 在 git 仓库里随 pull 自动同步；Knowledge 在 Devin 账号里需手动重建一次（读 devin-control/knowledge/ 按 frontmatter 重建，_autogen 跳过）；central-memory-safe 需手动重连。
""",
  metadata={"type":"decision","project":"2026ip"}
)
```

完成后用 `recall(projects=["2026ip"])` 验证能查到本条。
